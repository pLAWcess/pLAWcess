## 6. 회원 차단(`blocked`) 시 동작 정책

로그인 시 `account_status === 'blocked'` 면 **403** + 안내 메시지를 응답한다 (`apps/api/src/app/api/auth/login/route.ts:65-69`). 비밀번호가 맞아도 차단 계정은 세션이 발급되지 않는다.

기존 세션 즉시 무효화는 JWT stateless 특성상 비용 문제로 보류 — 차단 시점에서 최대 토큰 만료까지 잔여 세션이 유효할 수 있다.

## 7. 어드민 회원 상세 — 데이터 저장 위치

어드민이 회원 상세에서 수정한 값이 어디 저장되는지 명시:

| UI 필드 | 저장 위치 |
|---------|----------|
| 이름·생년월일·성별·연락처·이메일·학번·전공·학부 학교·입학/졸업년도·계정상태·역할 | `User` 테이블 |
| 학적상태 | `current_role === 'mentee'` → 가장 최근 `MenteeRecord`<br>`current_role === 'mentor'` → 가장 최근 `MentorRecord` |
| 소속 로스쿨·로스쿨 기수 | 가장 최근 `MentorRecord` |

→ 신청서(`*Record`)가 한 건도 없는 회원은 학적상태·로스쿨 정보를 저장할 곳이 없음.
**어드민 PATCH는 record 없으면 silent skip.** (회원이 신청서 제출하면 비로소 입력 가능)
