## 파일별 변경 내용

### 1. `apps/web/src/lib/api.ts`

기존 파일 끝(qualitative 섹션 뒤)에 `#176 Admin API` 섹션 추가. BE 응답 카멜케이스에 맞춰 모든 타입을 카멜로 정의.

**추가된 타입**
```ts
type Paged<T> = { data: T[]; totalCount: number; page: number; limit: number };
type AdminAccountStatus = 'active' | 'inactive' | 'blocked';
type ApplicationStatusLabel = 'pending' | 'approved' | 'rejected' | 'revision';
type AdminMenteeRow / AdminMentorRow
type AdminMenteeApplicationRow / AdminMentorApplicationRow
type EligibleMentee / EligibleMentor / EligiblePool
type AnnouncementRow
type AdminUserDetail / AdminUserGender / AdminUserAcademicStatus / AdminUserCurrentRole
type AdminUserParticipation / PatchAdminUserBody
```

**추가된 함수**
| 함수 | 엔드포인트 |
|---|---|
| `getAdminUsers(role, page?, limit?)` | `GET /api/admin/users?role=&page=&limit=` |
| `getAdminApplications(role, options?)` | `GET /api/admin/applications?role=&year=&page=&limit=` |
| `patchAdminApplication(id, { status?, memo? })` | `PATCH /api/admin/applications/:id` |
| `getEligibleMatchingPool(year?)` | `GET /api/admin/matchings/eligible?year=` |
| `listAdminAnnouncements(page?, limit?)` | `GET /api/admin/announcements` |
| `createAnnouncement({ title, body })` | `POST /api/admin/announcements` |
| `deleteAnnouncement(id)` | `DELETE /api/admin/announcements/:id` |
| `listAnnouncements(page?, limit?)` | `GET /api/announcements` (공개) |
| `getAnnouncement(id)` | `GET /api/announcements/:id` (공개) |
| `getAdminUser(userId)` | `GET /api/admin/users/:userId` |
| `patchAdminUser(userId, body)` | `PATCH /api/admin/users/:userId` |

오버로드 시그니처를 사용해 `role='mentee'` / `'mentor'` 별로 결과 타입이 정확히 좁혀지도록 처리. 페이지네이션 쿼리 빌더 `buildPaging` 과 공통 에러 처리 `jsonOrError` 헬퍼 동봉.

### 2. `apps/web/src/app/admin/users/page.tsx` — 회원관리 목록

**제거**
- mock `MENTEES`, `MENTORS` 배열 (총 19행)
- `Mentee`, `Mentor` mock 타입 정의

**변경**
- `MemberPanel` → `UserListPanel<T>` 로 교체. role 별 데이터 fetch 를 `useEffect` 안에서 처리.
- 클라이언트 페이지네이션 → **서버 페이지네이션** (page state 가 변경되면 API 재호출).
- `totalCount` 를 응답에서 받아 `totalPages = Math.ceil(totalCount / PAGE_SIZE)` 계산.
- 검색·정렬·status 필터는 **현재 페이지 결과 한정** (서버에선 페이지만, 추가 필터링은 FE).
- `account_status` enum 에 `blocked` 추가 — 필터 옵션·badge 색상 모두 반영.
- 멘티 컬럼 의미 명확화: `college/major` → `firstMajor/secondMajor` (제1전공/제2전공). 멘토 cohort 표시 `${cohort}기` 포맷.
- loading / error 상태 표시.
- 이름 셀의 Link 가 `/admin/users/${m.userId}` (snake → camel) 로 변경.

### 3. `apps/web/src/app/admin/users/[userId]/page.tsx` — 회원 상세

**제거**
- mock `USERS` 객체 (5명 hardcoded), `Gender`/`AcademicStatus`/`CurrentRole`/`AccountStatus` 로컬 타입 정의
- `LAWSCHOOLS`, `COHORTS` 옵션 배열, `genderFromLabel` 외 변환 헬퍼 일부

**변경**
- 페이지 진입 시 `getAdminUser(userId)` 로 서버에서 1회 fetch. 로딩/오류/404 분기.
- `ProfileCard` 의 저장 → `patchAdminUser` 호출, 응답으로 부모 state 갱신.
- `AccountCard` 도 동일하게 `patchAdminUser({ accountStatus, currentRole })`.

- 참여 이력은 BE 가 Application 합성으로 반환한 배열 그대로 표시.

이 화면이 사용하는 BE 라우트는 `apps/api/src/app/api/admin/users/[userId]/route.ts` (이번 PR 에 함께 추가).

## 응답 형식 정책 (FE 적용 결과)

### 모든 응답 카멜케이스
- 기존 mock 의 `user_id`, `student_id` 등 snake 필드는 모두 `userId`, `studentId` 로 교체.
- `accountStatus`, `applicationId`, `createdAt`, `submittedAt`, `lawSchool`, `cohort` 등 일관 적용.

### 배열형 응답 페이지네이션 래퍼
- `users`, `applications`, admin/공개 `announcements` 의 응답이 모두 `{ data, totalCount, page, limit }` 객체.
- FE 는 `setRows(res.data)` + `setTotalCount(res.totalCount)` 패턴으로 받음.
- `eligible` 만 단일 객체 (`{ year, mentees, mentors }`) — 페이지네이션 미적용.
